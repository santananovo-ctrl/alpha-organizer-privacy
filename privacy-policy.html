01

Quem somos

O Alpha Organizer Ops é uma extensão para o Google Chrome desenvolvida pela Assessoria Alpha, destinada exclusivamente a uso interno pela equipe operacional para gerenciar grupos de clientes no WhatsApp Web.

Esta Política de Privacidade descreve como coletamos, usamos, armazenamos e protegemos os dados tratados pela extensão, em conformidade com a Lei Geral de Proteção de Dados (LGPD — Lei nº 13.709/2018) e com as políticas de desenvolvedor do Google Chrome Web Store.

02

Dados coletados

A extensão coleta e processa as seguintes categorias de dados:

💬 Mensagens do WhatsApp Web

Textos de conversas lidas nos grupos acessados pelo usuário autenticado, para análise e categorização operacional.

🏷️ Dados de clientes

Status, fase de contrato, saúde do cliente (health score), datas de renovação e anotações operacionais.

🔑 Credenciais de acesso

Logins e senhas de plataformas externas (Meta Ads, Instagram, iFood, sistema de cardápio) inseridos manualmente pelo operador.

📊 Dados de desempenho (Meta)

Saldo e informações de contas do Meta Ads obtidos via API, com token fornecido pelo operador.

🤖 Análises de IA

Capturas de mensagens enviadas à API da Anthropic (Claude) para geração de resumos e insights automáticos.

⚙️ Configurações

Preferências da extensão, chaves de API configuradas pelo operador e configurações do Firebase.

Importante: A extensão não coleta dados de usuários finais (clientes dos franqueados). Os dados tratados referem-se exclusivamente a informações operacionais gerenciadas pelos membros internos da equipe Alpha.
03

Como usamos os dados

Os dados são utilizados exclusivamente para as seguintes finalidades:

  • Organizar e categorizar grupos de clientes no WhatsApp Web
  • Registrar e acompanhar ações, tarefas e histórico operacional por cliente
  • Monitorar métricas de saúde do cliente (health score, satisfação, fase)
  • Gerar análises automáticas de conversas via IA (Claude / Anthropic)
  • Consultar saldo de campanhas do Meta Ads em tempo real
  • Sincronizar dados operacionais entre dispositivos via Firebase Firestore
  • Autenticar operadores com controle de sessão e limite de tentativas
04

Armazenamento dos dados

Os dados são armazenados em dois ambientes:

  • Localmente no navegador (chrome.storage.local): registros operacionais, capturas de IA, configurações e sessão de autenticação. Esses dados permanecem no dispositivo do operador e não são transmitidos automaticamente a terceiros.
  • Firebase Firestore (Google Cloud): dados operacionais sincronizados entre membros da equipe, hospedados no projeto alpha-organizer-ops sob infraestrutura da Google com criptografia em trânsito (TLS) e em repouso.
Os dados armazenados no Firebase estão sujeitos à Política de Privacidade do Google (policies.google.com/privacy). Nenhum dado é transferido para servidores próprios da Assessoria Alpha além do Firebase configurado.
05

Compartilhamento com terceiros

A extensão integra-se com os seguintes serviços externos. Dados são enviados a eles apenas quando o operador utiliza a funcionalidade correspondente:

  • Anthropic (Claude API) — trechos de mensagens do WhatsApp são enviados para geração de análises e resumos. Aplicam-se os Termos de Uso da Anthropic.
  • Meta / Facebook Graph API — o token do operador é utilizado para consultar saldo de conta de anúncios. Nenhum dado é armazenado além do cache temporário.
  • Google Firebase — sincronização de dados entre operadores autorizados da mesma equipe Alpha.

Não vendemos, alugamos nem compartilhamos dados com quaisquer outras empresas ou serviços fora dos listados acima.

06

Permissões da extensão

A extensão solicita as seguintes permissões do Chrome:

Permissão Finalidade
storage Salvar registros operacionais, capturas de IA e configurações localmente no navegador.
tabs Detectar quando o usuário está na aba do WhatsApp Web para ativar o painel operacional.
https://web.whatsapp.com/* Injetar o painel de gestão e capturar mensagens dos grupos ativos no WhatsApp Web.
*.googleapis.com / *.firebaseio.com Comunicação com o Firebase (Firestore e Authentication) para sincronização e autenticação.
api.anthropic.com Enviar mensagens selecionadas para análise de IA (Claude) quando a funcionalidade estiver habilitada.
graph.facebook.com Consultar saldo de conta Meta Ads com o token fornecido pelo operador.
07

Segurança

Adotamos as seguintes medidas de segurança:

  • Autenticação por senha com bloqueio automático após 5 tentativas incorretas (lockout de 5 minutos)
  • Sessões com tempo de expiração de 8 horas
  • Comunicação com APIs externas exclusivamente via HTTPS/TLS
  • Chaves de API armazenadas apenas no chrome.storage.local do dispositivo do operador
  • Dados no Firebase protegidos por regras de segurança do Firestore
08

Seus direitos (LGPD)

Nos termos da Lei Geral de Proteção de Dados, os titulares têm os seguintes direitos:

  • Confirmar a existência de tratamento de dados pessoais
  • Acessar os dados armazenados sobre si
  • Corrigir dados incompletos, inexatos ou desatualizados
  • Solicitar a anonimização, bloqueio ou eliminação de dados desnecessários
  • Solicitar a portabilidade dos dados a outro fornecedor
  • Revogar o consentimento a qualquer momento

Para exercer esses direitos, entre em contato pelo canal indicado na seção de contato abaixo.

09

Menores de idade

Esta extensão é destinada exclusivamente ao uso profissional por membros adultos da equipe interna da Assessoria Alpha. Não coletamos intencionalmente dados de menores de 18 anos.

10

Alterações nesta política

Reservamo-nos o direito de atualizar esta política a qualquer momento. Alterações relevantes serão comunicadas via atualização da extensão na Chrome Web Store ou por canais internos da equipe Alpha.

A data de "última atualização" no topo desta página indica quando a versão vigente foi publicada. O uso continuado da extensão após alterações implica aceitação da nova política.
